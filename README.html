Setup Guide

  1. Download the app ZIP and unzip anywhere on your computer.
  2. Open index.html in your browser (Chrome/Edge recommended).
  3. Click “Load Sample” to explore, or import your own CSV.
  4. Map columns under Column Mapping if auto-detect missed anything.
  5. Set your Contribution Margin % and click Fit Curve.

CSV Requirements

  • Ad Spend is required. Ad Sales and/or Total Sales recommended.
  • If Ad Sales (or Total) are missing, provide ROAS (or TROAS) and the app will compute them as Spend × ROAS.
  • Include a Date column if you want the “Current spend” to be based on the latest row.

Gemini Integration (Optional)

You can generate a plain-English summary using Gemini:

  1. Enter your Gemini API Key and Model (e.g., gemini-2.5-flash or gemini-2.5-pro).
  2. Recommended: run the local proxy to avoid browser CORS: 
    cd server
npm install
node proxy.js
    This starts a server on http://localhost:8787. The app calls /server/gemini which proxies to Gemini.

How it Works

The app fits two common saturating response curves (Hill and Exponential) to your data via a robust Nelder–Mead optimizer, optionally with a baseline when Total Sales are present. It then finds the profit-maximizing spend where margin × mROAS = 1. We also compute an elbow (“knee”) as a visual diminishing-returns marker.

Outputs

  • Optimal spend, ad sales, ROAS, mROAS, profit
  • Current vs optimal headroom for spend, ROAS, ad sales, profit
  • R², elasticity at current and optimal
  • Interactive charts with hover readouts
